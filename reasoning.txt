DFS performs poorly here because your ghost is solving a moving-target chase problem with replanning every tick, and DFS is the wrong search bias for that setup.

Main reasons:

1. DFS is not shortest-path
It returns the first path it finds, not the best one. In a maze chase game, that often means long detours before reaching Pac-Man.

2. DFS is highly order-dependent
Neighbor expansion order in the transition graph drives behavior. Small ordering effects can produce very different paths, including awkward back-and-forth decisions.

3. You replan every frame against a moving goal
Pac-Man moves each step, so the DFS path computed at time \(t\) is stale at \(t+1\). With DFS’s unstable first-found behavior, this can cause oscillation between nearby cells.

4. DFS gives weak “chase pressure”
In pursuit, you want consistent gradient toward the target. BFS, Greedy, and especially A* provide that better:
- BFS: shortest in unweighted grid
- Greedy: fast, target-directed
- A*: target-directed + path-cost aware (usually best overall here)

5. Your implementation uses full recomputation
In IAAProject3/agents.py, DFS is recomputed each decision cycle via compute_path, which amplifies the instability in dynamic scenarios.

So DFS is not “wrong”; it is just mismatched to this task. It is better for exhaustive traversal, not real-time pursuit in a changing environment.


BFS Best???
It usually is not better overall. It is better only in specific conditions.

When BFS is better:
1. Uniform-cost moves and small map size.
2. You care about guaranteed shortest path length.
3. You want stable, predictable behavior independent of heuristic quality.
4. Your heuristic for Greedy or A* is weak/misleading for the map structure.

When Greedy is better:
1. You want fast decisions.
2. “Good enough” paths are fine.
3. You can tolerate non-optimal paths and occasional detours.

When A* is better:
1. You want shortest paths like BFS, but faster on larger maps.
2. You have a decent heuristic (Manhattan is usually good in grids).
3. You want a strong balance of speed and path quality.

So the key point:
- BFS: safest baseline, optimal in unweighted grids, but explores a lot.
- Greedy: fastest, but can be quite wrong.
- A*: usually the best practical choice if the heuristic is reasonable.

If you are seeing unstable chase behavior, BFS can appear “better” because it is deterministic and shortest-path correct, 
while Greedy can be erratic and A* can also look unstable if you fully replan every tick against a moving target.
